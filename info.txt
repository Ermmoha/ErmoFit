Ты — senior Android developer. Создай мобильное приложение “ErmoFit” для тренировок.

СТЕК И АРХИТЕКТУРА
- Kotlin, Android, Jetpack Compose, Material 3
- MVVM + Repository, Coroutines, Flow
- Navigation Compose (single-activity)
- DI: Hilt
- Локальная БД: Room
- Firebase Auth: email/password (регистрация/вход/выход)
- Firestore: привязанное к пользователю избранное + профиль
- DataStore Preferences: история поиска (последние 5 запросов)
- Media: Coil для изображений + ExoPlayer (Media3) для видео (и гиф через Coil)

ГЛАВНОЕ ТРЕБОВАНИЕ
Справочные данные (категории, упражнения, программы, связи программа->упражнения) должны храниться в Room.
Избранное должно привязываться к текущему Firebase user (uid) и храниться в Firestore.
Приложение должно собираться и запускаться без внешних платных ключей. Firebase подключение через google-services.json (пользователь добавит сам).

ДАННЫЕ: ПАРСИНГ -> ROOM
Сделай механизм загрузки данных в Room:
1) Основной вариант: загрузить JSON по HTTP (Retrofit/OkHttp) с какого-нибудь публичного raw URL (например GitHub raw). 
2) Фолбэк: если сеть недоступна или URL недоступен — загрузить тот же JSON из assets (assets/ermofit_seed.json).
3) При первом запуске (или если БД пустая) — распарсить JSON и сохранить в Room в транзакции.
4) Добавь версионирование seed: в DataStore хранить seedVersion; если версия JSON изменилась — обновить БД (стратегия: очистить и перезаполнить или аккуратная миграция — выбери и объясни).

JSON должен содержать:
- categories[]
- exercises[] (с mediaUrl, mediaType=image|gif|video)
- programs[] (с backgroundImageUrl, description, level, durationMinutes)
- programExercises[] (programId, exerciseId, order, defaultDurationSec, defaultReps)

MEDIA (ФОТО / GIF / ВИДЕО)
- В ExerciseDetailsScreen показывать медиа:
  - если mediaType=image или gif: показать через Coil AsyncImage
  - если mediaType=video: встроенный видеоплеер на базе Media3 ExoPlayer (autoplay off, кнопки play/pause)
- Для демо используй стабильные публичные URL (можно примеры с Wikimedia/peertube/mp4). Но обязательно: если ссылка не загрузилась — показать placeholder.

ТАЙМЕР И ВЫПОЛНЕНИЕ ТРЕНИРОВКИ
Нужно реализовать выполнение программы с таймером:
- Экран WorkoutPlayerScreen(programId)
- Показывает упражнения по порядку (из связующей таблицы programExercises)
- Для каждого упражнения:
  - название, описание/подсказка, медиа (image/gif/video)
  - таймер:
    - если упражнение по времени (durationSec > 0) — обратный отсчет
    - если упражнение по повторам (reps > 0) — таймер необязателен, но добавь “секундомер отдыха” между подходами (например 30 сек)
  - кнопки: Start/Pause, Next, Previous, Finish
  - прогресс: “Упражнение 2 из 8”, прогрессбар
  - опционально: RestScreen на 15–30 секунд между упражнениями (настраиваемо)
- Таймер должен переживать поворот экрана и пересоздание Compose (состояние в ViewModel). Для long-running — достаточно ViewModel + savedStateHandle; фоновые сервисы не обязательны, но обработай onDispose корректно.

ROOM СХЕМА
Сущности:
- CategoryEntity(id, title, imageUrl?)
- ExerciseEntity(id, title, description, muscleGroup, equipment, tags(List<String> as TypeConverter), mediaType, mediaUrl)
- ProgramEntity(id, title, description, level, durationMinutes, categoryId, backgroundImageUrl)
- ProgramExerciseEntity(programId, exerciseId, orderIndex, defaultDurationSec, defaultReps) (primaryKey: programId+exerciseId)
DAO:
- getProgramsRecommended(limit)
- getProgramById
- getExercisesForProgram(programId) (join + order)
- searchPrograms(query)
- searchExercises(query)
- getProgramsByCategory(categoryId)
- getByIds(list)
TypeConverters для List<String>

Структура (выбери 1 и реализуй):
